Conditional statements

  • We need to “tell” R when to do something.
  • This can be done by injecting logic into the code (TRUE/FALSE)
  • Conditional statements are powerful because we can have R execute different commands with a simple block of code.

The if statement

The most commonly-used conditional statement is the if statement:

if (condition) {
  Execute this code if condition is TRUE
}

In English: “If this condition is met, then do this”

if (condition) {
  Execute this code if condition is TRUE
} else {
  Execute this code if condition is FALSE
}

Nesting if statements

You can also nest several if statements:

if (condition) {
  Execute this code if condition is TRUE
  if (condition2) {
    Execute this code if condition2 is TRUE
  } else {
    Execute this code if condition2 is FALSE
  }
} else {
  Execute this code if condition is FALSE
}

How do we tell R what the condition we want met is?

Functions

We can define a function so that when we call it (with a specific value), we will get the output.

myFunction <- function(a = 0, b = 0) {
if (!is.numeric(a) || !is.numeric(b)) {
  output <- "a is not a number and/or b is not a number"
} else if (a > 0 && b > 0) {
  output <- "Both a and b are positive"
} else if (a < 0 && b < 0) {
  output <- "Both a and b are negative"
} else if (a == 0 || b == 0) {
  output <- "a is 0 and/or b is 0"
} else {
  output <- "a and b have different signs"
}
return(output)
}

Simulating \(\pi\)

  • We can use a simulation to estimate to area under the curve.
  • This is known as a Monte Carlo simulation.
    • Algorithms that enable us to calculate probabilities/integrals that may be hard to do analytically.
  • Calculating the proportion of “hits” (green dots) will give us an estimate of the area under the quarter arc. \[ p = \frac{\mbox{# of green dots}}{\mbox{Total # of dots}}\]
  • A green dot is a coordinate pair \((x, y)\) where \(x^2 + y^2 \leq 1\).
  • The area of the quarter arc (of a unit circle) is \(\pi / 4\).
  • We can estimate \(\pi\) by \(4 \times p\).

piSimulator <- function(N = 100) {
  x <- runif(N, min = 0, max = 1)
  y <- runif(N, min = 0, max = 1)
  
  hit <- 0
  for(i in 1:N) {

    } else {
      hit
    }
  }
  result <- 4 * sum(hit) / N
  return(result)
}

Exercise 2: The Pass Line

  • Craps is a dice game in which players bet on the outcomes of a pair of dice.
  • Easiest bet is the pass line bet:
    • If the come-out (first) roll is 7 or 11, the bet wins.
    • If the come-out roll is 2, 3 or 12, the bet loses.
    • If the roll is any other value, it establishes a point
      • If, with a point established, that point is rolled again before a 7, the bet wins.
      • If, with a point established, a 7 is rolled before the point is rolled again, the bet loses.
  • The probability of winning the pass line is \(\approx 49\%\).
  • We can try and program a function to confirm this.
